# AI 错题集项目需求文档

版本：v0.1  
项目代号：AI Error Book  
目标架构：Monorepo，React Web + Go API Server，后续扩展微信小程序

## 1. 项目背景

学生在日常练习、考试、作业订正中会产生大量错题，但传统错题整理依赖手工抄题、手工分类、手工复习，成本高且难以长期坚持。本项目 MVP 先通过 Web 网页端完成拍照/图片上传录入、AI OCR 识别、知识点自动归类、记忆曲线推荐、自动批改与答疑，形成一套从“错题采集”到“复习闭环”的智能错题管理系统。微信小程序作为后续迭代实现。

项目核心不是简单存题，而是帮助用户持续消化错题：自动识别错在哪里、属于哪个知识点、什么时候该复习、复习后是否真正掌握。

## 2. 产品目标

1. 降低错题录入成本：用户通过 Web 网页端拍照或上传图片即可生成结构化错题。
2. 提升错题整理质量：AI 自动识别题目、答案、解析、学科、年级、知识点。
3. 建立复习闭环：网页端每日推荐错题，用户作答后自动批改和答疑。
4. 自动维护掌握状态：连续两次推荐答对后，错题自动归入已完成。
5. 建立可维护的教材知识点库：管理员可按省份、年级、学科录入教材知识点，MVP 优先支持广东省。
6. 支持长期扩展：后端保留多端接入、多人账户、多省份教材、不同 AI 服务商和知识库扩展能力。

## 3. 用户角色

### 3.1 学生用户

主要使用者，负责拍照录入错题、查看错题集、完成每日一题、查看 AI 批改和答疑。

### 3.2 家长用户

可查看学生错题、掌握情况、薄弱知识点、复习完成度。MVP 阶段可先与学生账户合并，后续扩展亲子账户绑定。

### 3.3 管理员

负责用户管理、AI 调用配置、题目识别异常排查、系统数据统计和内容安全审查。

管理员还负责维护各省教材知识点库，包括省份、年级、学科、教材版本、章节目录和知识点标签。MVP 阶段先录入广东省常用教材知识点，但后台和数据模型必须保留其它省份的录入能力。

## 4. 产品端划分

### 4.1 Web 网页端

定位：MVP 阶段的主入口，覆盖错题采集、管理、复习和答疑。

核心任务：

1. 提供用户登录界面。
2. 提供管理员登录界面。
3. 拍照上传错题。
4. 支持相册选择图片。
5. 展示 AI 识别结果。
6. 允许用户手动修正题目、答案、学科、知识点。
7. 允许用户确认或修改 AI 识别出的学科和年级。
8. 提交并保存到错题集。
9. 查看最近录入状态和识别失败原因。
10. 查看错题列表。
11. 搜索和筛选错题。
12. 查看错题详情。
13. 每日一题复习。
14. 在线作答。
15. AI 自动批改。
16. AI 答疑。
17. 查看已完成错题和学习统计。

### Web 页面布局

Web 端采用经典 Codex app 风格的左中右三栏布局，作为 MVP 的主工作台布局。

1. 左侧栏：显示菜单功能项和主要导航，包括错题录入、错题列表、每日一题、统计看板和管理员入口。
2. 中间主区域：显示当前任务的核心内容。在每日一题和错题详情中，中间区域用于展示题目、题目图片、解析入口、答题大输入面板、数学表达式快捷输入栏和提交按钮。MVP 答题区使用 MathLive，整块面板均可键盘输入并支持公式编辑；按 Enter 应支持脱式运算式的多行书写。
3. 右侧栏：作为草稿区，供用户在答题时记录思路、草稿步骤和临时计算。MVP 草稿区同样使用 MathLive，整块右侧草稿面板均可键盘输入，支持文字、数字和 LaTeX 数学表达式；按 Enter 应支持换行记录推导过程。

布局要求：

1. 左侧菜单宽度稳定，便于快速切换功能。
2. 中间题目区是视觉和交互重心，答题输入框不能被右侧草稿区挤压。
3. 右侧草稿区可折叠，窄屏下可切换为抽屉或底部面板。
4. 管理员后台可以复用三栏骨架，但菜单项和中间内容应切换为后台管理能力。
5. 电脑端答题输入框应方便输入数学表达式，MVP 提供 MathLive 大输入面板和常用 LaTeX 快捷按钮，如分数、根号、上标、下标、积分、求和、π、θ 和约等于。
6. 每日一题页面不应出现页面级滚动条；题目区、功能区和提交按钮保持固定高度，中间答题输入框使用 flex 自动占满剩余空间，答题输入框和草稿输入框内容超出时只在各自面板内部滚动。
7. 富文本数学编辑能力以独立 Demo 页面先行验证：TinyMCE 5 负责富文本编辑，MathLive 负责公式录入，MathJax 负责将 LaTeX 渲染为 SVG 公式图片；公式图片需保留 `data-latex` 以支持双击回填编辑，并在最终预览区保持一致渲染。

### 4.2 微信小程序端

定位：后续迭代的移动端录入入口。

MVP 阶段暂不实现微信小程序。后续小程序能力与 Web 录入能力保持一致，包括拍照上传、识别结果确认、学科年级修改和错题保存。

### 4.3 Go 后端服务

定位：业务核心和 AI 编排中心。

核心任务：

1. 用户认证和权限控制。
2. 图片上传和文件管理。
3. OCR 与题目结构化识别。
4. 知识点归类。
5. 错题 CRUD。
6. 记忆曲线推荐。
7. 自动批改和答疑。
8. 学习状态更新。
9. 数据统计和日志审计。

## 5. 核心业务流程

### 5.1 错题录入流程

1. 用户在 Web 网页端拍照或选择图片。
2. Web 上传图片到后端。
3. 后端保存原始图片，并创建识别任务。
4. 后端调用 OCR 和结构化服务识别题目内容。
5. AI 返回结构化建议结果：题干、选项、用户答案、正确答案、解析、学科、年级、知识点。
6. 后端保存 AI 初始识别结果和置信度。
7. Web 展示识别结果，用户可手动修正题目、答案、学科、年级和知识点。
8. 用户确认后，系统以用户最终确认的学科和年级重新匹配教材知识点。
9. 错题正式入库，并自动归入用户的默认错题本。
10. 系统保存 AI 建议值、用户确认值和默认错题本归属。
11. 系统初始化该错题的复习状态。

### 5.2 错题管理流程

1. 用户打开 Web 错题集页面。
2. 系统按学科、知识点、掌握状态、创建时间展示错题列表。
3. 用户可通过关键词搜索题干、解析、知识点。
4. 用户可进入错题详情页查看图片、识别文本、答案解析、复习记录。
5. 用户可编辑错题信息或删除错题。

### 5.3 每日一题流程

1. 用户进入每日一题页面。
2. 后端根据记忆曲线、错题状态、上次复习时间、错误次数、知识点薄弱度推荐一道错题。
3. 用户提交答案。
4. 后端调用 AI 批改服务判断答案是否正确，并生成简短反馈。
5. 若答错，系统生成答疑说明，并更新错题复习状态。
6. 若答对，系统记录一次正确复习。
7. 当同一道错题连续两次被推荐且均答对，系统将其归入已完成类别。

### 5.4 AI 答疑流程

1. 用户在错题详情或每日一题结果页发起提问。
2. 后端携带题目、正确答案、解析、用户答案、历史问答上下文调用大模型。
3. AI 返回分步骤讲解。
4. 系统保存问答记录，供用户后续查看。

## 6. 功能模块需求

## 6.1 用户与认证模块

### 功能列表

1. 用户注册和登录。
2. Web 微信 OAuth 登录。
3. 获取用户手机号。
4. Session 鉴权。
5. 用户资料维护。
6. 管理员用户管理。
7. 注册时选择省份和年级。
8. 支持用户后续修改省份和年级。
9. 管理员登录。
10. 管理员 Session 鉴权。

### 功能说明

Web 端需要提供两个登录入口：普通用户登录页和管理员登录页。普通用户登录页建议路径为 `/login`，用于学生用户进入错题本系统；管理员登录页建议路径为 `/admin/login`，用于进入后台管理系统。

MVP 阶段普通用户只支持微信 OAuth 登录。用户首次登录时必须完成微信授权，并获取用户手机号；手机号用于账户绑定、身份识别和后续找回能力。用户注册时还必须录入省份和年级，系统后续会将该信息作为默认教材地区和默认识别上下文。系统需要保证不同用户之间的数据隔离。后端接口必须基于用户身份过滤错题、复习记录和答疑记录。

普通用户每次登录成功后创建服务端 Session，Session 有效期为 30 天。30 天内用户访问 Web 端无需重复登录；超过有效期后需要重新走微信 OAuth 登录流程。

管理员登录与普通用户微信 OAuth 登录分离。管理员可使用后台专用账号密码登录，登录成功后创建管理员 Session，有效期同样为 30 天。管理员账号不得开放公开注册，只能通过初始化脚本、后台种子数据或已有管理员创建。

省份和年级属于核心学习画像字段，不应只作为展示资料保存。注册省份用于确定默认教材地区，注册年级用于默认识别上下文；具体到单道错题时，AI 知识点归类、错题筛选、薄弱项统计和每日一题推荐应优先使用用户最终确认的错题学科和错题年级。

### MVP 范围

1. 支持 Web 端微信 OAuth 登录。
2. 支持登录时获取并绑定用户手机号。
3. 支持服务端 Session，单次登录有效期 30 天。
4. 支持基础用户信息：昵称、头像、省份、年级。
5. 注册时省份默认可选广东省，同时保留其它省份选项和后台数据扩展能力。
6. 支持管理员登录页和管理员专用 Session。

## 6.2 图片上传与 OCR 模块

### 功能列表

1. 拍照上传。
2. 相册上传。
3. 图片压缩。
4. 图片裁剪。
5. 图片存储。
6. OCR 识别。
7. 识别任务状态查询。
8. 识别失败重试。

### 功能说明

Web 端负责采集图片，后端负责统一保存图片和触发识别流程。识别流程应异步执行，避免上传接口长时间阻塞。OCR 默认采用百度开源 PaddleOCR 方案，负责从错题图片中提取文本、公式附近文本和版面区域信息。识别结果需要保留原始 OCR 输出和后续 AI 结构化输出，便于排查和二次解析。

OCR 与题目结构化应解耦。PaddleOCR 负责图片到文本，后端再将 OCR 文本、图片区域信息、用户省份和年级、教材知识点候选集交给结构化服务，生成题干、答案、解析和知识点匹配结果。

### 关键状态

1. `uploaded`：图片已上传。
2. `processing`：AI 正在识别。
3. `recognized`：识别完成。
4. `confirmed`：用户已确认。
5. `failed`：识别失败。

## 6.3 错题结构化模块

### 功能列表

1. 题干提取。
2. 选项提取。
3. 用户答案识别。
4. 正确答案识别。
5. 解析提取。
6. 学科识别。
7. 年级识别。
8. 题型识别。
9. 难度评估。
10. 置信度评分。
11. 学科和年级确认。

### 结构化字段

1. 题目标题。
2. 题干正文。
3. 题目图片。
4. 题型：选择题、填空题、解答题、判断题、作文题、其他。
5. 学科：数学、语文、英语、物理、化学、生物、历史、地理、政治、其他。
6. 年级。
7. 学期。
8. 正确答案。
9. 用户原答案。
10. AI 解析。
11. 知识点标签。
12. 错因标签。
13. AI 建议学科。
14. AI 建议年级。
15. 用户确认学科。
16. 用户确认年级。

### 学科和年级确认规则

AI 识别出的学科和年级是建议值，不是最终值。Web 确认页必须允许用户修改学科和年级，因为不同年级学生可能提前学习更高年级内容，也可能补学低年级内容。

用户注册时填写的年级用于缩小默认识别范围和推荐候选知识点，但错题最终归类、教材知识点匹配、统计分析和复习推荐应以用户确认后的错题学科和错题年级为准。

## 6.4 知识点归类模块

### 功能列表

1. 自动识别知识点。
2. 支持多知识点标签。
3. 支持用户手动增删标签。
4. 支持知识点树。
5. 支持按知识点统计错题数量。
6. 支持薄弱知识点排序。
7. 按省份、年级、学科匹配教材知识点。
8. 支持无法匹配时生成候选知识点，供管理员审核入库。

### 功能说明

AI 需要根据题目内容、答案和解析识别知识点。系统应保留“教材知识点”“系统知识点”和“用户自定义标签”三类标签。教材知识点用于精准统计和复习推荐，系统知识点用于跨地区通用归类，自定义标签用于个人整理。

知识点匹配时应优先使用用户所在省份，以及用户最终确认的错题学科和错题年级。例如广东省初二学生上传初三数学内容时，AI 可先参考注册年级给出建议，但用户确认该题为初三数学后，系统应在“广东省 + 初三 + 数学”的教材知识点树中重新匹配。若无法匹配到明确节点，AI 可以生成候选知识点，但候选知识点需要标记为待确认，不直接进入正式教材知识库。

### 知识点示例

1. 数学：一元二次方程、函数图像、几何证明、概率统计。
2. 英语：时态、从句、完形填空、阅读理解。
3. 物理：力学、电学、光学、热学。
4. 化学：化学方程式、酸碱盐、氧化还原。

## 6.5 教材知识点库模块

### 功能列表

1. 管理省份。
2. 管理年级。
3. 管理学科。
4. 管理教材版本。
5. 管理章节目录。
6. 管理知识点树。
7. 批量导入知识点。
8. 手动新增、编辑、禁用知识点。
9. 审核 AI 生成的候选知识点。
10. 查询某省某年级某学科的知识点树。

### 功能说明

教材知识点库是错题自动归类的基础数据。管理员需要能够按“省份 + 年级 + 学科 + 教材版本”维护知识点。MVP 阶段优先录入广东省，但系统不能把广东省写死在代码中，应通过数据表和管理后台配置。

知识点树建议分为四层：

1. 学科。
2. 教材册别或学期。
3. 章节。
4. 具体知识点。

### MVP 范围

1. 默认内置或后台录入广东省。
2. 支持新增其它省份，但不要求前期完整录入。
3. 支持广东省小学、初中、高中常用年级枚举。
4. 支持管理员手动录入知识点树。
5. 支持 CSV 或表格粘贴式批量导入。
6. 支持知识点启用、禁用和排序。

## 6.6 错题集管理模块

### 功能列表

1. 错题列表。
2. 错题详情。
3. 错题编辑。
4. 错题删除。
5. 错题搜索。
6. 错题筛选。
7. 错题状态管理。
8. 已完成错题归档。
9. 默认错题本归属。

### 错题本规则

MVP 阶段先只使用默认错题本。每个用户默认拥有一个系统默认错题本，用户录入错题时不需要选择错题本，所有新错题自动归入默认错题本。

错题本是用户个人收纳维度，知识点是教材学习维度。MVP 暂不开放自定义错题本、错题本重命名和错题移动能力，但数据模型保留 `Notebook`，为后续扩展多错题本做准备。

### 筛选维度

1. 学科。
2. 年级。
3. 知识点。
4. 题型。
5. 掌握状态。
6. 创建时间。
7. 最近复习时间。
8. 错误次数。

### 掌握状态

1. `new`：新录入，未复习。
2. `learning`：学习中。
3. `reviewing`：复习中。
4. `mastered`：已完成。
5. `archived`：用户手动归档。

## 6.7 每日一题模块

### 功能列表

1. 获取今日推荐题。
2. 展示题目内容。
3. 在中间主区域提供在线作答输入框。
4. 提交答案。
5. 自动批改。
6. 展示反馈。
7. 查看解析。
8. 发起答疑。
9. 跳过当前题。
10. 在右侧草稿区记录答题思路。
11. 支持通过快捷按钮向答题输入框插入常用 LaTeX 数学表达式。

### 推荐规则

系统推荐错题时综合考虑：

1. 距离上次复习时间。
2. 历史错误次数。
3. 连续答对次数。
4. 知识点薄弱程度。
5. 错题创建时间。
6. 题目难度。
7. 用户所在省份、错题确认学科和错题确认年级对应的教材知识点优先级。

### 完成规则

同一道错题在两次推荐中均答对，且两次推荐不是同一天刷出来的重复答题，则自动进入 `mastered` 状态。

## 6.8 记忆曲线推荐模块

### 功能列表

1. 初始化复习计划。
2. 计算下次复习时间。
3. 根据答题结果动态调整间隔。
4. 每日推荐优先级排序。
5. 支持补复习。

### 初始推荐间隔

MVP 可采用简化艾宾浩斯间隔：

1. 第 1 次：录入后当天或次日。
2. 第 2 次：1 天后。
3. 第 3 次：3 天后。
4. 第 4 次：7 天后。
5. 第 5 次：14 天后。

答错时缩短间隔，答对时拉长间隔。连续两次答对后进入已完成。

## 6.9 自动批改模块

### 功能列表

1. 对比标准答案。
2. 支持主观题语义批改。
3. 判断正确、部分正确、错误。
4. 生成批改理由。
5. 识别用户主要错误点。
6. 保存批改记录。
7. 生成本次作答的知识点诊断明细。

### 批改结果

1. `correct`：正确。
2. `partial`：部分正确。
3. `wrong`：错误。
4. `uncertain`：AI 不确定，需要用户自行确认。

### 约束

自动批改默认使用 DeepSeek Flash 模型。AI 批改必须输出理由和置信度。若置信度过低，不应强行判定，应提示用户手动确认。

批改模型需要通过配置项管理，默认值为 DeepSeek Flash，后续可按成本、稳定性或题型切换到其它兼容大模型。

批改完成后，系统需要将本次作答结构化保存为 `AnswerRecord`，并将暴露出的知识点问题保存为 `AnswerKnowledgeDiagnosis`。错题详情页展示历史作答记录时，应能看到每次答题是否正确、错误原因、关联知识点、严重程度和 AI 解释。

## 6.10 AI 答疑模块

### 功能列表

2. 根据用户问题追问答疑。
3. 结合用户答案指出错误原因。
4. 给出同类题解题思路。
5. 保存问答历史。

### 答疑风格

1. 面向学生，语言通俗。
2. 分步骤讲解。
3. 不直接输出过长内容。
4. 优先解释为什么错，而不是只给答案。
5. 可根据省份、年级和教材知识点调整表达难度。

### 功能说明

统计模块服务于 Web Dashboard，帮助用户快速知道当前学习状态。MVP 阶段以错题数量、复习数量、掌握状态、知识点分布为主。

## 9. AI 能力需求

### 9.0 默认模型选型

1. OCR 默认方案：百度开源 PaddleOCR。
2. 自动批改默认模型：DeepSeek Flash。
3. AI 答疑默认模型：DeepSeek Flash。
4. 知识点归类默认模型：DeepSeek Flash。
5. 模型调用层必须保留 Provider 抽象，允许后续切换 OpenAI、Gemini、DeepSeek 或其它兼容接口。

PaddleOCR 主要负责图片文字识别，不直接承担题目理解、答案批改和答疑。DeepSeek Flash 主要负责文本结构化后的语义判断、知识点匹配、自动批改和答疑生成。

### 9.1 OCR 与结构化识别 Prompt 目标

PaddleOCR 需要先从图片中提取可用文本和版面信息，结构化服务再基于 OCR 输出和用户学习画像生成稳定 JSON：

1. 题干。
2. 选项。
3. 学科。
4. 年级。
5. 省份适配信息。
6. 题型。
7. 正确答案。
8. 用户答案。
9. 解析。
10. 知识点。
11. 置信度。

识别时应将用户注册资料中的省份和年级传入上下文，作为默认识别范围。同时要求 AI 输出建议学科和建议年级，并允许后续由用户确认或修改。用户确认后，系统需要按确认后的学科和年级重新执行教材知识点匹配。

### 9.2 知识点归类 Prompt 目标

AI 需要基于题目和解析生成知识点标签，并尽量匹配用户所在省份、用户确认年级、用户确认学科下的教材知识点。不能匹配时，可生成候选标签，由用户或管理员后续确认。

### 9.3 自动批改 Prompt 目标

AI 需要基于题目、标准答案、解析和用户答案返回 JSON：

1. 是否正确。
2. 部分正确程度。
3. 批改理由。
4. 错误点。
5. 置信度。
6. 建议复习知识点。
7. 本次作答的知识点诊断列表。

1. 用户数据隔离。
2. 图片访问需鉴权。
3. AI Key 不暴露给前端。
4. 敏感日志脱敏。

## 11. 后端服务模块划分

### 11.1 拆分原则

后端按业务边界拆分 Service，避免把每个接口、每个数据库表或每个小动作都拆成独立服务。Service 的粒度应以“能独立表达一组业务能力、方便测试、方便维护”为准。

拆分时遵循以下原则：

1. 一个 Service 负责一个清晰业务域。
2. 同一业务域内的增删改查、状态流转和核心校验尽量放在同一个 Service 中。
3. Handler 只负责参数解析、鉴权上下文、调用 Service 和返回响应。
4. Repository 只负责数据读写，不承载业务决策。
5. AI、OCR、文件存储等外部能力通过 Provider 或 Client 抽象接入。
6. 不按单个函数拆服务，避免出现大量 `CreateXService`、`UpdateXService` 这类过细结构。

### 11.2 核心 Service 列表

#### UserService

负责用户与认证相关能力：

1. 微信 OAuth 登录。
2. 手机号绑定。
3. 首次注册资料补全。
4. Session 创建、刷新、校验和退出。
5. 用户资料读取和更新。
6. 管理员用户管理。

#### ErrorItemService

负责错题主业务：

1. 错题确认入库。
2. 错题列表、详情、编辑和删除。
3. 默认错题本归属。
4. 错题状态流转。
5. 错题搜索和筛选。

#### RecognitionService

负责图片识别和题目结构化：

1. 图片上传后的识别任务创建。
2. 调用 OCR Provider。
3. 调用结构化模型生成题目字段。
4. 保存 AI 建议学科、年级、知识点和置信度。
5. 识别失败重试和状态查询。

#### CurriculumService

负责教材和知识点库：

1. 省份、年级、学科管理。
2. 教材版本管理。
3. 章节和知识点树管理。
4. 广东省教材知识点录入。
5. AI 候选知识点审核。

1. 记忆曲线计算。
2. 每日一题推荐。
3. 复习记录创建。
4. 连续答对次数更新。
5. 已完成状态判断。

#### GradingService

负责自动批改和知识点诊断：

1. 调用 DeepSeek Flash 批改用户答案。
2. 生成 `AnswerRecord`。
3. 生成 `AnswerKnowledgeDiagnosis`。
4. 更新错题掌握状态。
5. 返回批改结果、讲解和知识点诊断。

#### ChatService

负责 AI 答疑：

1. 基于错题、解析、用户答案和历史上下文生成答疑。
2. 保存答疑消息。
3. 控制答疑上下文长度。

#### StatsService

负责统计分析：

1. 错题总数、待复习数、已完成数。
2. 学科分布。
3. 知识点薄弱榜。
4. 正确率趋势。
5. 复习完成度。

#### AdminService

负责管理员聚合能力：

1. 管理员权限校验。
2. 用户管理聚合接口。
3. AI 调用日志查看。
4. OCR 失败任务查看。
5. 系统配置和模型配置。

管理员端涉及教材知识点维护时，具体业务仍由 `CurriculumService` 承载，`AdminService` 只做权限和后台聚合编排。

## 13. MVP 范围

### 必须实现

1. Web 拍照或图片上传。
2. AI 识别错题。
3. 用户确认识别结果。
4. Web 错题列表。
5. Web 搜索筛选。
6. 每日一题推荐。
7. 用户作答。
8. AI 自动批改。
9. 连续两次答对自动完成。
10. 基础统计看板。
11. 用户通过微信 OAuth 注册登录，并获取手机号。
12. 管理员录入广东省教材知识点。
13. 知识点归类优先匹配用户省份、错题确认学科和错题确认年级。
14. OCR 默认接入百度开源 PaddleOCR。
15. 自动批改默认接入 DeepSeek Flash。
16. 新录入错题自动归入默认错题本。
17. 每次登录 Session 有效期 30 天。
18. Web 端提供用户登录界面和管理员登录界面。
19. Web 主工作台采用左中右三栏布局。

### 暂不实现

1. 家长账户绑定。
2. 班级和老师管理。
3. 付费订阅。
4. 大规模题库同类题生成。
5. 自定义错题本和多人协作错题本。
6. 复杂学习报告导出。
7. 全国所有省份教材知识点一次性完整内置。
8. 微信小程序端。
9. 普通用户账号密码登录、邮箱登录、手机号验证码登录。

## 14. 验收标准

1. 用户可从 Web 上传或拍摄一张错题图片，并在 30 秒内看到结构化识别结果。
2. 用户可手动修正 AI 识别出的题目、答案、学科、年级和知识点并保存。
3. Web 端可查看该错题，并能按学科和知识点搜索。
4. 每日一题页面能推荐一条待复习错题。
5. 用户提交答案后，系统能返回批改结果、讲解和本次作答的知识点诊断。
6. 同一道错题连续两次推荐答对后，状态自动变为已完成。
7. 统计页能展示错题总数、待复习数、已完成数和薄弱知识点。
8. 新用户必须通过微信 OAuth 登录并授权手机号，同时选择省份和年级，否则不能完成注册。
9. 管理员能录入广东省某年级某学科的教材知识点，并在错题识别后被用于知识点匹配。
10. 当用户把 AI 识别的年级修改为其它年级时，系统应按修改后的年级重新匹配教材知识点。
11. 用户确认录入后，错题应自动归入默认错题本。
12. 用户登录成功后 Session 有效期为 30 天，30 天内刷新或重新打开 Web 端应保持登录态。
13. Web 端必须提供普通用户登录页和管理员登录页，普通用户登录页走微信 OAuth，管理员登录页进入后台专用登录流程。
14. 管理员账号密码登录成功后应进入后台管理页，且管理员密码不得以明文形式存储。
15. 每日一题页面应采用三栏布局：左侧菜单，中间显示题目和答题大输入面板，右侧显示草稿大输入面板；答题区和草稿区在 MVP 均使用 MathLive，整块 panel 均可键盘输入。
